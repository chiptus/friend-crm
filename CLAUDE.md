# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- **Dev server:** `pnpm dev` (runs on port 3000)
- **Build:** `pnpm build`
- **Preview prod build:** `pnpm preview`
- **Run tests:** `pnpm test` (Vitest with jsdom)
- **Run single test:** `pnpm test -- src/path/to/test.test.ts`
- **Deploy:** `pnpm deploy` (builds + deploys to Cloudflare Workers)
- **Generate types:** `pnpm cf-typegen` (regenerates `worker-configuration.d.ts`)
- **DB generate migration:** `pnpm db:generate` (drizzle-kit generate)
- **DB migrate local:** `pnpm db:migrate:local` (apply migrations to local D1)
- **DB migrate prod:** `pnpm db:migrate:prod` (apply migrations to remote D1)
- **DB studio:** `pnpm db:studio` (Drizzle Studio GUI)

## Architecture

**Fullstack TypeScript monolith** using React 19 + TanStack Router + TanStack React Start, deployed to **Cloudflare Workers** with **Cloudflare D1** (edge SQLite) and Drizzle ORM.

### Key directories

- `src/routes/` — File-based routing. Files here auto-generate `src/routeTree.gen.ts` (read-only, don't edit).
- `src/server/` — Backend: database schema (`schema.ts`), D1 database helper (`db.ts`), server functions (`functions.ts`).
- `scripts/` — Utility scripts (migration guide).
- `drizzle/` — Generated migration SQL files.
- `public/` — Static/PWA assets.

### Data flow pattern

1. **Route loaders** call server functions to fetch data before render
2. **Server functions** (`createServerFn` from `@tanstack/react-start`) handle all server-side logic with Zod validation
3. **Mutations** call server functions then `router.invalidate()` to refetch data
4. **Database batches** ensure atomicity (e.g., logging interaction + updating lastContactedAt via `db.batch()`)

### Database

Cloudflare D1 (edge SQLite). Three tables: `friends`, `interactions`, `pushSubscriptions`. Schema defined with Drizzle in `src/server/schema.ts`. D1 binding accessed via `getDb()` from `src/server/db.ts`. All DB operations are async.

### Cloudflare

- **Runtime:** Cloudflare Workers via `@cloudflare/vite-plugin`
- **Config:** `wrangler.jsonc` — worker name, D1 bindings, compatibility flags
- **Types:** `worker-configuration.d.ts` — generated by `pnpm cf-typegen`
- **D1 access:** `import { env } from 'cloudflare:workers'` in server code

### Path aliases

`@/*` maps to `./src/*` (configured in both tsconfig.json and vite.config.ts).

### Styling

Tailwind CSS v4 via `@tailwindcss/vite` plugin. Single import in `src/styles.css`.

### PWA

Configured via `vite-plugin-pwa` with auto-update service worker and manifest in vite.config.ts.
